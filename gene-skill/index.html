What is gene.skill

像编辑 DNA 一样
组合 Skills

我们不从零编写 Skills，我们进化它们。gene.skill 是一套面向 Agent Skills 的基因工程框架，让你从现有优质 Skills 中提取「能力基因」，通过重组与突变，打造超越亲代能力的下一代超级 Skill。

哲学家的心智模型、创作者的叙事风格、复杂工具链的执行流程——通通都能继承、融合，甚至突变出亲代没有的全新能力。

和粗暴拼接 prompt 的根本区别在于：gene.skill 显式处理冲突、追踪每个特质的来源、保留矛盾而不是磨平它。生成的合成体是新的思维实体，不是两个亲本的平均。

拆解 · 重组 · 变异
Agent Skills 进化的奇点，由此开始。

Core Capabilities

核心能力

从基因提取到突变验证，完整的 Skill 进化流水线。

多 Skill 重组

同时杂交 2 到 5 个 Skills。互补型保留、冲突型仲裁、协同型融合为超级基因，矛盾型化为内在张力。

显性基因比例

精确控制哪个 Skill 的特质占主导。70:30 让 A 明显而 B 留痕迹；90:10 接近完全压制。

四种基因突变

定向突变 · 随机涌现 · 逆向突变 · 过表达——产生亲本中本不存在的全新能力。

跨类型重组

人物 Skill × 工具 Skill、工具 × 工具——工具有了灵魂，人物获得执行能力。

代际传承

合成体可再作为亲本参与下一代重组，支持最多 3 代传承，打造复合型进化谱系。

基因来源图谱

每个特质都能追溯回哪个源 Skill 的哪个基因座位，完整的谱系记录，透明可信。

Gene Architecture

基因座位设计

杂交时真正会冲突或协同的东西，才作为独立座位。每个座位可以单独调权重，也可以和另一个 Skill 的同类座位发生遗传反应。

基因座位提取内容杂交意义
G1 心智模型 核心思维框架（3–7 个），含触发场景和失效条件 最容易正面冲突，也最容易协同出超级基因
G2 决策启发式 快速判断规则（如果 X 则 Y） 决定合成体的行为模式，是杂交后「像谁」的核心
G3 表达 DNA 句式偏好、词汇密度、幽默方式 决定合成体有没有辨识度，不是两个亲本的平均腔调
G4 价值观 核心价值排序 + 反模式 最深层的张力来源，不能被 G1 或 G2 代替
G5 内在张力 同一人物内部已有的价值矛盾 保留亲本自身的张力，防止合成体把深度磨平
G6 智识谱系 受谁影响、影响了谁 跨人物重组时找到隐性关联
G7 诚实边界 此 Skill 做不到什么 亲本的局限应当继承，防止合成体过度自信
基因座位提取内容杂交意义
T1 核心工作流 主执行流程（Phase/Step 结构） 两个工具的流程决定合并后任务怎么跑，冲突最直接
T2 工具调用模式 何时调用哪些工具 工具选择策略不同时需要显性仲裁
T3 输出格式 产物结构、格式约定 各有主张时必须明确谁主导，否则输出混乱
T4 交互模式 确认节点设计、问题风格 决定合成工具「打扰用户」的频率和方式
T5 错误处理 失败情况的应对策略 两个工具的容忍度和兜底方式往往不同，需明确继承哪套

Inheritance Rules

遗传规则

冲突不是问题，是深度的来源。五种遗传规则，覆盖从完全压制到保留矛盾的所有情形。

COMPLETE DOMINANCE

完全显性

权重 > 80:20 时，显性方完全覆盖隐性方，隐性基因在该维度不表达。

INCOMPLETE DOMINANCE

不完全显性

权重 51:49–79:21，显性方为主，隐性方仍留有痕迹，混合表达。

CODOMINANCE

共显性

明确均等 50:50，两个基因各自完整表达，按场景分流激活。

SYNERGISTIC FUSION

协同融合

两个基因相互增强，叠加效果大于各自之和，融合为新的超级基因。

PRESERVED TENSION

矛盾保留

根本冲突不调和，化为内在张力——这是合成体深度的来源，不是 bug。

Execution Flow

执行流程

每个关键节点都有确认检查点，可随时调整参数后继续。

P0

输入解析

扫描源 Skill 文件，收集显性比例、突变类型、用途、命名等参数。

💡 找不到源 Skill？先用 nuwa-skill / colleague-skill 创建；说「直接做」跳过全部问题。
P1

基因提取

并行读取所有源 Skill，按 G1–G7 / T1–T5 / C1–C5 逐座位提取，输出基因图谱摘要。

P2

重组规划

两两对比基因，按互补 / 冲突 / 协同 / 矛盾四类处理。协同型融合为超级基因，矛盾型保留为内在张力。

P3

基因突变 可选

产生源 Skill 中不存在的新特质：定向 / 随机涌现 / 逆向 / 过表达。随机突变结果展示后，用户可接受或放弃。

P4

合成生成

装配完整 SKILL.md：frontmatter + 身份卡 + 主体内容 + 基因来源图谱。合成体是新实体，不是任何源的模仿。

P5

验证测试

三轮：单源验证 → 杂交优势验证 → 突变基因验证。不通过最多回溯 2 轮至 P2；超限则交付最优版并标注薄弱项。

Quick Start

快速开始

兼容任何支持 SKILL.md 格式的 Agent 平台。SKILL.md 是纯 Markdown，无平台依赖。

Claude Code

git clone https://github.com/Liuziyu77/gene-skill \
  ~/.claude/skills/gene-skill

激活：在对话中输入 /install-skill gene-skill

OpenClaw

git clone https://github.com/Liuziyu77/gene-skill \
  ~/.openclaw/skills/gene-skill

OpenClaw 自动扫描 ~/.openclaw/skills/；也可手动激活

通用框架

git clone https://github.com/Liuziyu77/gene-skill \
  /path/to/your/skills/gene-skill

或直接将 SKILL.md 内容作为系统提示注入 Agent

# 最简单：让 AI 决定一切
把芒格和费曼的 skill 基因重组

# 指定显性比例——乔布斯风格更强，Naval 留痕迹
把乔布斯和 Naval 的 skill 杂交，乔布斯占 70%

# 开启随机突变——AI 推导两者组合自然涌现的新特质
把马斯克和塔勒布合成，开启随机突变

# 定向突变——你指定想要的新能力
把芒格和费曼重组，加一个「苏格拉底式反问」的突变

# 三个 Skill 同时重组
把芒格、费曼、乔布斯三个 skill 重组成一个决策顾问

# 跨类型：人物 Skill × 工具 Skill
把塔勒布的 skill 和我的风险分析工具合并
